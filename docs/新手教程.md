# RAGcore 新手教程

面向**第一次使用** RAGcore 的用户：从下载安装，到导入笔记、配置 AI、开始问答。

---

## RAGcore 是什么？

RAGcore 是一款**个人笔记知识库**桌面软件：

- 在本地保存你的 Markdown / 文本 / PDF 笔记
- 用本地 AI 模型做语义检索（不需要把笔记上传到云端）
- 问答时调用 **DeepSeek** 大模型，根据你的笔记内容生成回答

一句话：**把你的笔记变成可对话的私人知识库。**

---

## 第一步：获取软件

你会收到下面两种形式之一（由发布者提供）：

| 形式 | 文件名示例 | 你怎么做 |
|------|------------|----------|
| **安装包（推荐）** | `RAGcore-win-x64-setup.exe` | 双击安装 → 从开始菜单或桌面打开「RAGcore」 |
| **绿色版压缩包** | `RAGcore-win-x64.zip` | 解压到任意文件夹 → 进入文件夹双击 `RAGcore.exe` |

### 重要说明

- **不能只发 `RAGcore.exe` 一个文件**。exe 只是启动器，必须和 `python`、`backend`、`frontend` 等文件夹在一起。
- 整个安装目录约 **2–3 GB**（含本地 AI 模型），解压或安装需要几分钟。
- 首次打开时 Windows 可能提示「已保护你的电脑」——点「更多信息」→「仍要运行」即可（未签名软件常见提示）。

### Mac 用户

下载 `RAGcore-mac-arm64.zip`（Apple 芯片）或 `RAGcore-mac-x64.zip`（Intel），解压后将 `RAGcore.app` 拖入「应用程序」，双击打开。

---

## 第二步：第一次启动

1. 双击 **`RAGcore.exe`**（Windows）或 **RAGcore 图标**（Mac）
2. 会弹出 **「RAGcore 笔记知识库」状态窗口**，表示后台服务正在运行
3. 浏览器会自动打开：**http://127.0.0.1:8765**
4. **不要关闭状态窗口**——关闭即停止服务；最小化即可

若浏览器没自动打开，在状态窗口点「打开浏览器」，或手动访问上述地址。

---

## 第三步：注册账号

RAGcore 是**本地多用户**系统，数据存在你自己的电脑上：

1. 在网页点 **「注册」**
2. 填写用户名、邮箱、密码
3. 注册成功后登录

> 账号仅用于本机访问控制，不会上传到互联网。

---

## 第四步：配置 DeepSeek API Key（必做）

RAG 检索在本地完成，但**生成回答**需要联网调用 DeepSeek API。

### 4.1 获取 API Key

1. 打开 [DeepSeek 开放平台](https://platform.deepseek.com/)
2. 注册 / 登录
3. 进入「API Keys」创建一个新 Key
4. 复制 Key（形如 `sk-...`），妥善保存

### 4.2 在 RAGcore 里填写

1. 登录后，左侧进入 **「设置」**
2. 模型名称保持默认 **`deepseek-v4-flash`**（推荐，速度快）
   - 需要更强推理可改 **`deepseek-v4-pro`**（更慢、更贵）
   - 旧名 `deepseek-chat` 将于 2026-07-24 停用，请勿再使用
3. API 地址保持：`https://api.deepseek.com/v1/chat/completions`
4. 粘贴你的 **API Key**
5. 点 **「保存」**

保存成功后，masked Key 会显示为 `****xxxx` 形式。

---

## 第五步：导入笔记

1. 左侧进入 **「笔记」**
2. 可以：
   - **新建笔记**：在线写 Markdown
   - **导入文件**：批量上传 `.md`、`.txt`、`.pdf`
3. 导入后系统会在后台**自动切块、向量化**（右下角可看任务进度）

### 首次向量化较慢

第一次导入或首次问答时，需要加载本地 BGE 模型，大约 **30 秒～1 分钟**，属正常现象。之后会变快。

---

## 第六步：开始问答

1. 左侧进入 **「问答」**
2. 输入问题，例如：
   - 「我上周关于项目计划的笔记里提到了哪些里程碑？」
   - 「总结一下所有关于 Python 的笔记要点」
3. 回车或点发送
4. AI 会检索你的笔记片段，并给出带来源引用的回答

回答末尾会列出引用的笔记标题。若笔记中没有相关内容，会明确提示「未找到相关信息」。

---

## 数据存在哪里？

你的笔记和配置**不会**随软件卸载自动删除：

| 平台 | 数据目录 |
|------|----------|
| Windows | `%APPDATA%\RAGcore\` |
| Mac | `~/Library/Application Support/RAGcore/` |

其中包含：

- `storage/` — 笔记数据库与向量索引
- `.env` — API Key 等配置
- `logs/` — 运行日志

更新或重装软件时，只要不动这个目录，数据就还在。

## 日常使用习惯

| 操作 | 说明 |
|------|------|
| 开机后用 RAGcore | 双击 `RAGcore.exe` → 等状态窗口显示「服务运行中」 |
| 用完关闭 | 在状态窗口点「退出并停止服务」，或直接关窗口并确认 |
| 换电脑 | 复制整个 `%APPDATA%\RAGcore\` 数据目录到新电脑同路径（需重新安装软件） |
| 备份 | 定期备份 `%APPDATA%\RAGcore\storage\` |

## 常见问题

### 双击 exe 闪退 / 没反应

1. 确认是**整个文件夹**一起解压的，不是只复制了 exe
2. 查看日志：`%APPDATA%\RAGcore\logs\ragcore.err.log`
3. 检查 8765 端口是否被占用（关掉其他 RAGcore 实例）

### 网页打不开

- 等状态窗口从「正在启动」变为「服务运行中」（约 5–15 秒）
- 手动访问 http://127.0.0.1:8765

### 问答没反应 / 报错

- 检查「设置」里 API Key 是否正确、账户是否有余额
- 确认电脑能访问 `api.deepseek.com`
- 模型名应为 `deepseek-v4-flash` 或 `deepseek-v4-pro`

### 导入 PDF 失败

- 单文件不超过 10 MB（可在配置中调整）
- 扫描版 PDF 文字识别效果取决于 PDF 本身

### Windows 提示病毒 / 不安全

未签名桌面软件常见误报。若软件来自你信任的来源，可添加白名单；正式发布者可购买代码签名证书消除提示。

## 发布者：如何发给用户

若你是帮别人打包软件的人，请参考：

- 开发者环境：[启动说明.md](启动说明.md)
- Windows 打包：[packaging/win/README.md](../packaging/win/README.md)

```powershell

# 或安装包（需 Inno Setup）
iscc packaging\win\installer.iss
# → dist\RAGcore-win-x64-setup.exe
```

附上一份本教程或 `packaging/用户安装说明.txt` 即可。
