# RAGcore 启动与环境准备

本文说明**开发、打包、运行**前需要安装什么、下载什么，以及如何启动。

| 你是谁 | 需要装什么 | 怎么启动 |
|--------|------------|----------|
| 最终用户 | 无需 Python，下载桌面包即可 | [packaging/用户安装说明.txt](../packaging/用户安装说明.txt) |
| 开发者改代码 | Python、Node.js、模型文件 | 见下文「开发者启动」 |
| 维护者打 Windows 包 | 开发者依赖 + Git Bash 等 | [packaging/win/README.md](../packaging/win/README.md) |

## 一、开发者：要装哪些依赖

### 1.1 系统与基础软件

| 依赖 | 版本建议 | 用途 | 验证命令 |
|------|----------|------|----------|
| **Python** | 3.11+ | 后端、模型脚本 | `python --version` |
| **Node.js** | 18+ | 前端开发与构建 | `node -v` / `npm -v` |
| **Git**（含 Git Bash） | 最新 | 拉代码；下载 reranker 脚本 | `git --version` / `bash --version` |

Windows 上建议用 [python.org](https://www.python.org/downloads/) 安装 Python，勾选 **Add to PATH**。

### 1.2 Python 包（后端）

在项目 `backend` 目录创建虚拟环境并安装：

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\activate          # Mac/Linux: source .venv/bin/activate
pip install -r requirements.txt
```

主要依赖包括：FastAPI、uvicorn、PyTorch、sentence-transformers、ChromaDB、PyMuPDF 等。  
首次 `pip install` 会下载 PyTorch 等大包，**约 5–15 分钟**，属正常情况。

锁定版本（打包用）见 `backend/requirements-lock.txt`。

### 1.3 Node 包（前端）

```powershell
cd frontend
npm install
```

开发模式无需先 `npm run build`；使用 `scripts/start-win.bat` 一键启动时需要已构建的 `frontend/dist`。

### 1.4 本地模型文件（必下，否则无法向量化 / 打包）

RAG 使用两个本地 BGE 模型，需放到 `backend/models/`：

| 模型 | 目录 | 体积约 | 是否必须 |
|------|------|--------|----------|
| 嵌入模型 | `backend/models/bge-small-zh-v1.5/` | ~90 MB | **必须** |
| 重排序模型 | `backend/models/bge-reranker-base/` | ~1.1 GB | 建议（`.env` 默认开启） |

**嵌入模型**（Hugging Face，新版 CLI 用 `hf`）：

```powershell
# 在项目根目录执行
pip install -U huggingface_hub
hf download BAAI/bge-small-zh-v1.5 --local-dir backend/models/bge-small-zh-v1.5
```

> 提示 `huggingface-cli` 已废弃，请用 `hf download`，不要用旧命令。  
> 未登录时下载较慢（几十到几百 KB/s），**进度条在动就不是卡死**，大文件 `model.safetensors` 约 90MB，耐心等完即可。  
> 可选加速：在 [Hugging Face](https://huggingface.co/settings/tokens) 创建 Token 后执行 `hf auth login`。

校验：

```powershell
dir backend\models\bge-small-zh-v1.5\config.json
dir backend\models\bge-small-zh-v1.5\model.safetensors
```

**重排序模型**（ModelScope，脚本会自动装 `modelscope`）：

```powershell
# 需 Git Bash 或 WSL
bash backend/scripts/download_reranker.sh
```

校验：

```powershell
dir backend\models\bge-reranker-base\model.safetensors
```

### 1.5 配置文件

```powershell
cd backend
copy .env.example .env    # Mac/Linux: cp .env.example .env
```

编辑 `backend/.env`，至少配置：

```env
LLM_MODEL_API_KEY=你的-DeepSeek-API-Key
```

RAG 问答需联网调用 DeepSeek；本地检索不依赖 API Key。

## 二、开发者：怎么启动

### 方式 A — 前后端分开（改代码推荐）

**终端 1 — 后端：**

```powershell
cd backend
.\.venv\Scripts\activate
uvicorn server.main:app --reload --port 8000
```

**终端 2 — 前端：**

```powershell
cd frontend
npm run dev
```

浏览器打开：http://localhost:5173  
API 文档：http://127.0.0.1:8000/docs

### 方式 B — 一键启动（接近桌面包体验）

先构建前端一次：

```powershell
cd frontend
npm run build
```

再启动（需已配置 `backend/.venv`）：

```powershell
scripts\start-win.bat        # Windows
# scripts/start-mac.command  # Mac
```

浏览器：http://127.0.0.1:8765

## 三、维护者打 Windows 包：额外依赖

在**第二节全部就绪**的基础上，还需：

| 依赖 | 用途 |
|------|------|
| 嵌入模型已下载 | `build-win.ps1` 会检查 `config.json`，缺了直接报错 |
| Git Bash | 缺 reranker 时脚本调用 `download_reranker.sh` |
| Inno Setup 6（可选） | 生成 `setup.exe` 安装包 |
| 磁盘 ≥ 8 GB | 构建临时文件 + 约 3–4 GB 产物 |

```powershell
powershell -ExecutionPolicy Bypass -File packaging\win\build-win.ps1
```

完整说明：[packaging/win/README.md](../packaging/win/README.md)

## 五、常见问题

**`hf download` 很慢，像卡住了？**  
大文件在后台下载，速度取决于网络。只要进度在增加就继续等；可 `hf auth login` 提高限额。

**`缺少 backend\models\bge-small-zh-v1.5`（打包报错）？**  
先完成上文 1.4 嵌入模型下载，再重新执行 `build-win.ps1`。

**`/health` 能打开，但问答没反应？**  
检查设置页 DeepSeek API Key、网络，以及嵌入 / reranker 模型是否齐全。

**`scripts/start-win.bat` 提示未就绪？**  
按 1.2 创建 `backend/.venv` 并 `pip install -r requirements.txt`，再 `npm run build` 前端。

**日志在哪？**  
- 开发者：`backend/storage/` 或终端输出  
- 桌面包：`%APPDATA%\RAGcore\logs\`
